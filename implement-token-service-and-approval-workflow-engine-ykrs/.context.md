# Goal
Build the approval workflow engine that manages request lifecycle and issues tokens.

# Acceptance Criteria
- [ ] Request state machine (pending → approved/denied/expired)
- [ ] Approval policy engine with path-based rules
- [ ] JWT token generation and validation
- [ ] Token revocation capability
- [ ] Request expiration and cleanup
- [ ] Multi-approver support with quorum

# Files to Create
- internal/workflow/engine.go (approval workflow)
- internal/workflow/state.go (state machine)
- internal/workflow/policy.go (policy evaluation)
- internal/token/service.go (JWT generation)
- internal/token/validator.go (JWT validation)
- internal/scheduler/cleanup.go (expired request cleanup)

# Key Implementation Details
- JWT signed with RS256 (RSA private key)
- Short-lived tokens (configurable TTL)
- Policy rules: required_approvals, approver_groups, max_duration
- Background job to cleanup expired requests
- WebSocket support for real-time status updates (future)
